# 初音ミク Figure Archive
### Complete Hatsune Miku figure database — scraped daily from MyFigureCollection.net

---

## Project Structure

```
miku-db/
├── scraper/
│   ├── mfc_scraper.py          ← Python scraper (Beautiful Soup)
│   └── requirements.txt
├── src/
│   └── MikuDB.jsx              ← React frontend
├── public/
│   ├── figures.json            ← Auto-generated by scraper (gitignored initially)
│   └── meta.json               ← Scrape stats & timestamp
├── .github/
│   └── workflows/
│       └── daily_scrape.yml    ← GitHub Actions cron (3 AM SGT daily)
└── README.md
```

## Quick Start

### 1. Run the scraper locally (first-time full scrape)

```bash
cd scraper
pip install -r requirements.txt

# Full scrape (all figures, takes ~30–60 min due to polite rate limiting)
python mfc_scraper.py

# Incremental (only new figures since last run — fast, used by daily cron)
python mfc_scraper.py --incremental

# Test with just 3 pages
python mfc_scraper.py --limit 3
```

This produces `public/figures.json` and `public/meta.json`.

### 2. Run the frontend

```bash
# Using Vite (recommended)
npm create vite@latest . -- --template react
npm install
# Copy src/MikuDB.jsx → src/App.jsx
npm run dev
```

Or just open `MikuDB.jsx` directly in Claude.ai as an artifact.

## GitHub Actions Setup (Daily Cron)

### Step 1: Push to GitHub
```bash
git init
git add .
git commit -m "feat: initial miku figure archive"
git remote add origin https://github.com/YOUR_USERNAME/miku-db
git push -u origin main
```

### Step 2: Enable Actions
The workflow at `.github/workflows/daily_scrape.yml` runs automatically:
- **Schedule**: Every day at **3:00 AM Singapore Time** (19:00 UTC)
- **Mode**: Incremental (only fetches figures not already in `figures.json`)
- **Commit**: Auto-commits updated `figures.json` if anything changed

### Step 3: (Optional) Deploy to Vercel
```bash
npm install -g vercel
vercel --prod
```

Then add your Vercel deploy webhook as a repo secret:
- `Settings → Secrets and variables → Actions`
- Add secret: `VERCEL_DEPLOY_HOOK` = your Vercel deploy hook URL

## Hatsune Miku on MFC

- **Character ID**: `2156`
- **Browse URL**: `https://myfigurecollection.net/browse.v4.php?type=-1&characters=2156`
- The scraper paginates through all pages (48 items/page) and enriches each stub with its detail page

## Filter Capabilities

| Filter         | Options                                                      |
|---------------|--------------------------------------------------------------|
| Figure Type    | Scale, Nendoroid, Pop Up Parade, Figma, Prize, Mini, etc.   |
| Series / Event | Snow Miku, Racing Miku, Magical Mirai, Symphony, Append…    |
| Illustrator    | KEI, Rella, iXima, Fuzichoco, GEMI, Saine, Ontama…          |
| Manufacturer   | All makers (dynamic dropdown)                                |
| Year Range     | 2007 → present                                              |
| Max Price      | JPY ceiling                                                  |
| Sort           | Newest, Oldest, Name, Maker, Price                           |
| View           | Grid (cards) or List (rows)                                  |

---

## Notes

- MFC requests are rate-limited to **1.5s between pages** and **0.4s between detail pages** out of respect for the site
- The scraper gracefully falls back to stub data if a detail page fails
- The frontend shows demo data if `figures.json` is not yet available
- Click any figure card to open a full detail modal with all specs + MFC link
