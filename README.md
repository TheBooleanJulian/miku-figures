## What it does

Miku Figures is a searchable, filterable archive of every Hatsune Miku figure listed on MyFigureCollection.net. A Python scraper (BeautifulSoup) runs daily via GitHub Actions at 3 AM SGT, enriching each figure with detail-page data and writing the results to `public/figures.json`. The React + Vite frontend reads that static JSON and lets collectors browse, filter, and sort the full catalogue without a backend server.

## Features

- Full figure catalogue scraped from MFC, updated daily via GitHub Actions cron
- Incremental scraping mode вЂ” only fetches figures not already in the dataset, keeping runs fast
- Filter by figure type, series/event, illustrator, manufacturer, year range, and max price (JPY)
- Sort by newest, oldest, name, manufacturer, or price
- Grid and list view modes
- Rich data schema per figure: scale, price, dimensions, material, MFC owned count, illustrator, tags, and more
- Deployed as a static site (Vite build served via Nginx in Docker)

## Quick Start

### Run the scraper

```bash
cd scraper
pip install -r requirements.txt

# Full scrape вЂ” all figures (~30вЂ“60 min, polite rate limiting)
python mfc_scraper.py

# Incremental вЂ” only new figures since last run
python mfc_scraper.py --incremental

# Test run вЂ” 3 pages only
python mfc_scraper.py --limit 3
```

This writes `public/figures.json` and `public/meta.json`.

### Run the frontend

```bash
npm install
npm run dev
```

## Deployment

Frontend is deployed to **Vercel** at [miku-figures.vercel.app](https://miku-figures.vercel.app). A Vercel deploy webhook (`VERCEL_DEPLOY_HOOK` repo secret) triggers a redeploy automatically after the daily scrape commits updated JSON.

A **Dockerfile** is also provided for serving the Vite build via Nginx:

```bash
docker build -t miku-figures .
docker run -p 8080:80 miku-figures
```
